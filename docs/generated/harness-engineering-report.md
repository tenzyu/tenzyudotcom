# ハーネスエンジニアリング解析報告書：AIエージェント運用OSの現状と改善

## 1. 現状把握（Status Quo）

本プロジェクトにおける「ハーネス」は、物理的な配線ではなく、**「AIエージェントがリポジトリを自律運用するためのドキュメント駆動型OS」**として定義されています。

### 現行ハーネスの構成要素（BOM: 部品表）
- **主配線図 (Root Map)**: `AGENTS.md`（段階的開示の起点となる目次）
- **実行ユニット (Active Plans)**: `docs/exec-plans/active/*.md`（タスクごとの接続子）
- **接続仕様書 (Workflows)**: `docs/workflows/agent-orchestration-workflow.md`（委譲プロトコル）
- **検査装置 (Validator)**: `scripts/lint-docs.ts`（ドキュメント構造の自動検証機）
- **組立指示書 (Structure Rules)**: `docs/design-docs/structure-rules.md`（所有権・配置ルール）

### 接続状況の確認結果
- **過去の実績**: `completed/harness-orchestration-readiness.md` 等の記録から、過去には `execution-ready: true` フラグと `nix develop` コマンドを用いた高精度な自動委譲運用が正常に機能していたことを確認。
- **現在の稼働状態**: 現在 `active/` 配下にある9つのタスクはすべて「軽量なメモ（Follow-up）」であり、高負荷な自動実行モードではなく「ナレッジベースとしての待機モード」にあります。

---

## 2. 評価（Evaluation）

| 評価項目 | 判定 | 評価理由 |
| :--- | :---: | :--- |
| **信頼性** | **高** | `lint-docs.ts` による自動検証（リンク切れ、メタデータ欠落の検知）が強力に機能している。 |
| **安定性** | **中** | 軽量メモと実行計画が同じディレクトリに混在しており、委譲時の「インピーダンス不整合」リスクがある。 |
| **環境整合性** | **高** | `nix develop -c ...` による実行環境の固定がプロトコル化されており、環境差異による「ショート」を防いでいる。 |
| **保守性** | **中** | 所有権モデルが精緻なため、エージェントが「配置の慣性（Placement Smells）」に流されないための規律維持が必要。 |

---

## 3. 改善提案（Improvement Proposals）

現行ハーネスの潜在的なリスクを解消し、より強靭な「配線」を構築するための提案です。

### 提案1：アクティブ・プランの「ケーブル整理（整理整頓）」
- **現状**: `docs/exec-plans/active/` 内で、単なるメモと実行計画が混在し、視認性が低下している。
- **改善案**: 委譲準備が整っていないメモ書きプランに対し、ファイル名にプレフィックス（例: `note-*.md`）を付与するか、`active/backlog/` サブディレクトリへ分離する。
- **効果**: エージェントが「今、誰が、どこを、どのような精度で繋ごうとしているか」を一目で把握可能になる。

### 提案2：検証コマンドの「導通テスト（Dry-run）」強化
- **現状**: `execution-ready` プラン内の `Verification` コマンドは存在チェックのみで、内容の妥当性は未検証。
- **改善案**: `scripts/lint-docs.ts` を拡張し、`execution-ready: true` のプランに対して、記載された `nix develop -c ...` コマンドが文法的に正しいか、または実際に dry-run 可能かを検証する機能を追加する。
- **効果**: サブエージェントへ委譲した直後の「テスト実行エラー」による手戻り（断線）を未然に防ぐ。

### 提案3：エディトリアル・コネクタの最終固定
- **現状**: `vercel-blob-integration.md` がメモ状態のまま放置されており、ストレージ層の接続仕様が不安定。
- **改善案**: このタスクを `execution-ready: true` に昇格させ、`src/lib/editorial` における「Local fallback vs Storage override」の境界バリデーションを完成させる。
- **効果**: データフローの絶縁を確実にし、将来的な外部DB移行時のコストを削減する。

---

## 4. 総括
本プロジェクトのハーネスは、ソフトウェアの枠組みを超えた「エージェント運用インフラ」として極めて高い完成度にあります。現在のアクティブ・タスク群を「メモ」から「実行計画」へ昇格させる際の**「接続品質（Verificationの精度）」**を一段高めることで、さらなる自律運用の安定化が期待できます。
