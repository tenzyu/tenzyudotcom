# Design Principles (DESIGN)

このドキュメントは、このリポジトリにおけるUI/UXの設計原則を定義する**ゴールデンプリンシプル**である。エージェントはコードを書く際、以下の規律から逸脱してはならない。

## 1. 原則 (Premium & Dynamic Experience)

このリポジトリのUI/UXは、訪問者が一目で「プレミアム」だと感じる洗練された体験を提供することを目的とする。シンプルな Minimum Viable Product (MVP) のような見栄えではなく、ダイナミックで豊かなデザインを採用すること。

## 2. Token-first Styling (トークンファーストアプローチ)

- **マジックナンバーの禁止**: コンポーネントに直接 `px` 単位の数値をハードコードしたり、場当たり的な色（`#ff0000`など）を指定することは固く禁じられている。
- **デザインシステムの利用 (Tailwind CSS)**: 全てのスタイリングはTailwindCSSのユーティリティクラス（トークン）を用いて行う。`shadcn/ui` などの基盤コンポーネントが提供する汎用的なバリアント（`primary`, `secondary` 等）を再利用すること。
- **色彩とテーマ**: 汎用的な色（純粋な赤、青、緑）は避け、HSLベースの調和の取れたカラーパレットを使用する。サイト全体で洗練されたダークモードをサポートし、滑らかなグラデーションを活用すること。

## 3. Typography & Readability (タイポグラフィと可読性)

- **フォント**: デフォルトのブラウザフォントは避け、モダンなフォント（Inter, Roboto, Outfitなど）を利用する。
- **可読性の優先（ブログ/アーカイブ等）**: 長文コンテンツ（`/blog`など）では、読者の可読性を最優先にする。『Web Interface Guidelines』に従い、適切な行間（`leading`）、文字幅（`max-w-prose`）、文字サイズ（`text-base` からスケール）とコントラスト比を設定すること。
- **レイアウトの堅牢性**: 画像のレスポンシブな振る舞いや、長いコードブロックがコンテナを突き破らないためのオーバーフロー対策（`overflow-x-auto` 等）を必ず適用し、レイアウト崩れを防ぐこと。

## 4. Interaction & Animation (インタラクションとアニメーション)

- **ダイナミックな体験**: ユーザーの操作（ホバー、クリック）に対して、微細かつ心地よいマイクロアニメーション（Hover effects, Micro-animations）を実装する。
- **応答性**: コンポーネントは状態の変化に対して視覚的なフィードバックを即座に返し、サイトが生きているかのように感じさせる設計とすること。

## 5. Accessibility by Default (Web Interface Guidelines)

- **Semantic HTML**: 見た目のためだけに `<div>` や `<span>` を多用せず、目的に応じた正しいHTML要素（`<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<footer>`, `<button>` など）を使用すること。
- **SEOベストプラクティス**: 1ページにつき1つの `<h1>` を配置し、適切な階層構造 (`<h2>`, `<h3>`) を維持する。
- **aria属性とインタラクティブ要素**: 動的なインタラクションを伴うコンポーネントはWAI-ARIAガイドラインに従い、スクリーンリーダーやキーボードナビゲーションに対応させる。また、インタラクティブ要素にはブラウザテストやアクセシビリティツールのための一意で説明的な ID (`id`) を付与する。

## 6. Constraint over Creativity (創造性よりも制約)

LLMは、「新しくて派手なUI」を勝手に発明してはならない。すでにあるコンポーネントライブラリや `globals.css` のユーティリティを組み合わせることで成立するかを最初に検討し、どうしても表現できない場合のみ拡張を提案すること。
