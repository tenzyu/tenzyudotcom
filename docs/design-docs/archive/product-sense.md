# Product Sense (PRODUCT SENSE)

このドキュメントは、この個人サイトというプロダクトが「何を目指し、何を意図的にやらないか」を定義する**ゴールデンプリンシプル**である。エージェントは機能要件を考える際、常にこの価値基準と照らし合わせること。

## 1. Core Values (中核的価値)

このサイトの機能とコンテンツは、一時的な消費ではなく「Durable Memory（耐久性ある記憶）」と「Living Curation（生きたキュレーション）」を提供することを目標とする。単なるレジュメやポートフォリオではなく、以下の複合的価値を生み出すための「庭（Garden）」である。

- **Owned Identity (所有されたアイデンティティ)**:
  - 自分の名前、文体、制作物、導線をサードパーティのプラットフォームから自分自身のドメインに奪還し、集約すること。
- **Durable Memory (永続的な記憶)**:
  - タイムラインを流れて消えるSNSとは対極に、後から検索・見返しができる記録と導線を育てていくこと。
- **Living Curation (生きたキュレーション)**:
  - おすすめするもの（Recommendations）、使う道具（Tools）、気になるリンク（Links）など、書き手の「今」を更新し続けること。完成された過去だけではなく、現在進行形の興味を展示する。ブックマークや記事は単なるログではなく、文脈（Category/Description）を持たせ、読む人に意図を伝える構成（例: `/links`）とする。
- **Utility for Self (自己のための効用)**:
  - 訪問者だけでなく、サイトオーナー自身が毎日使うためのユーティリティ（Pointers、ダッシュボード等）を持つこと。

## 2. Feature Principles (機能の方針)

- **Feature-first, Pattern-later**:
  - 新機能の目的（Feature）を実現することを最優先し、アーキテクチャの美的整理（Pattern）はそのあとに続く。ユーザーへの価値提供が先である。
- **Durable Architecture**:
  - 流行のスタックに過度に依存せず、「退屈な技術（boring technology）」であっても、安定し、LLMエージェントが可読性を保ちやすく、数年後にも破綻しないアーキテクチャ構造を選択する。
- **Lightweight Admin Friction (摩擦のない更新)**:
  - 継続的な更新（Living Curation）を担保するためには、管理画面（Admin）からの更新フローが極めて軽く、スマホ等からでも簡単に記述・保存できなければならない。「URLを入れたらOGPが補完される」程度のミニマムな入力を重宝する。
- **Archive and Garden**:
  - 時系列で記事が流れる「Archive (Blogなど)」の側面と、カテゴリ別に常緑な情報が蓄積される「Garden (Links, Recommendationsなど)」の両面を両立させるシステムを持たせる。
- **Room for Unfinished Work**:
  - 「未完成の考え」や「短いメモ（Notes）」も置ける余白を残すこと。

## 3. Editorial Stance (編集・オーサリング)

コンテンツの作成と管理について、以下の分離と責務を明確にする。

- **`*.source.ts` (Canonical Source)**: 人間が編集・管理する根本のソースデータ。将来的にはCMSやDBなどのストレージに昇格しうるデータ本体。
- **`*.content.ts` (Meaning)**: UIのラベル、ローカライズされた文言、説明などの「意味」を保持するIntlayerの層。
- **分離の原則**: データ本体（識別子や安定したURL、レコード情報など）と、その見せ方・ラベル（Intlayer）は分離し、コンポーネント側（`*.assemble.ts`等）で結合して表示する。この構造を守ることで、多言語対応とコンテンツ更新の疎結合を実現する。

## 4. Non-Goals (目指さないもの)

以下の機能の導入は、プロダクトのコア価値から外れるため原則として棄却する。

- チームコラボレーションや複雑なアクセス権限（RBAC）
- 他者からのコメント機能や「いいね」機能によるソーシャルグラフの直接的持ち込み
- Fail-openによる一時的な状態の崩壊（更新が反映されないよりは、サイトがビルドエラーで落ちないことを優先するが、保存競合は後勝ちではなく適切にRejectする）
