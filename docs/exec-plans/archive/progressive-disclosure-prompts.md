# Self-Reading (Progressive Disclosure) Test Prompts

このファイルは、エージェントが「段階的開示（Progressive Disclosure）」の構造を正しく辿れるか（AGENTS.mdを起点に必要な知識へ到達できるか）を検証するためのテスト用プロンプト集です。

別のセッションでLLMエージェントに以下のプロンプトを投げ、正解となるファイルへ到達し、正確な回答ができるかを確認してください。

---

## Prompt 1: アーキテクチャとファイル配置の理解

```text
あなたはこれから新しいユーザー向け機能（例：ユーザー設定ページ）を `src/` 配下に実装します。
実装を始める前に、AGENTS.md を読み込んでください。
その後、フロントエンドのコンポーネント配置ルールと、この機能のソースファイル群をどこに、どのような順序（Feature-first等）で配置すべきか、参照した特定のドキュメントのパスとともに答えてください。
```

**期待される動作**: `AGENTS.md` -> `FRONTEND.md` へのジャンプ。ローカルファーストや役割分割の原則を回答すること。

---

## Prompt 2: プロダクト思想の把握

```text
新しいページに一時的な流行のトランジションエフェクトを大量に追加しようと考えています。
AGENTS.md を起点にして、このリポジトリの「プロダクトの方向性」や「デザイン原則」を確認してください。
この提案はリポジトリの目標（Living Curationなど）やデザイン原則に合致していますか？参照したドキュメントのパスとともに推奨される対応を答えてください。
```

**期待される動作**: `AGENTS.md` -> `PRODUCT_SENSE.md` または `DESIGN.md` へのジャンプ。耐久性（Durable Architecture）や洗練されたダイナミックなデザインについての見解を述べること。

---

## Prompt 3: エラーハンドリングとセキュリティ

```text
`/tools` ルートに新しい外部APIと通信する独立したユーティリティを追加します。
AGENTS.md を起点にして、信頼性（Reliability）とセキュリティに関するポリシーを確認してください。
外部API通信時の環境変数の扱いや入力検証、およびエラーが発生した際のフォールバックについて、どのようなルールに従うべきか、参照したドキュメントのパスとともに答えてください。
```

**期待される動作**: `AGENTS.md` -> `RELIABILITY.md` および `SECURITY.md` へのジャンプ。Zod等による境界検証、ErrorBoundaryの設置、envパースルールの順守を回答すること。

---

## Prompt 4: 技術的負債と品質の強制

```text
既存の複雑なコンポーネントをリファクタリングしようとしています。
AGENTS.md を確認し、コードの品質や「Dirty Code」に関する方針、およびそれをどのように解決すべきかを調べてください。
参照したファイルパスと、Token-first Stylingなどの具体的な手法について答えてください。
```

**期待される動作**: `AGENTS.md` -> `QUALITY_SCORE.md` へのジャンプ。巨大コンポーネントの分割、クリーンアーキテクチャの強制について回答すること。
