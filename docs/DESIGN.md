# Design Principles (DESIGN)

このドキュメントは、このリポジトリにおけるUI/UXの設計原則を定義する**ゴールデンプリンシプル**である。エージェントはコードを書く際、以下の規律から逸脱してはならない。

## 1. Token-first Styling (トークンファーストアプローチ)

- **マジックナンバーの禁止**: コンポーネントに直接 `px` 単位の数値をハードコードしたり、場当たり的な色（`#ff0000`など）を指定することは固く禁じられている。
- **デザインシステムの利用**: 余白、色、タイポグラフィの全ては、Tailwind CSS の設定（`tailwind.config.ts` またはプリセット）で定義されたデザイン・トークンを通して適用すること。
- **一貫性**: `shadcn/ui` などの基盤コンポーネントが提供する汎用的なバリアント（`primary`, `secondary`, `destructive`, `outline`, `ghost` など）を再利用し、新しい場当たり的なバリアントを作らないこと。

## 2. Accessibility by Default (アクセシビリティの標準化)

- **セマンティックなHTML**: 見た目のためだけに `<div>` や `<span>` を多用せず、目的に応じた正しいHTML要素（`<button>`, `<nav>`, `<article>`, `<main>` など）を使用すること。
- **aria属性の適切な利用**: 動的なインタラクションを伴うコンポーネント（例: モーダル、アコーディオン、ドロップダウン）は、WAI-ARIAガイドラインに従い、画面読み上げソフト（スクリーンリーダー）やキーボードナビゲーションに完全に対応していること。`shadcn/ui` (Radix UI) がこれを担保している場合は、その恩恵を破壊しないこと。
- **コントラストとカラー**: テキストカラーと背景色のコントラスト比を常に適正に保つこと。

## 3. Readability & Content Presentation (可読性とコンテンツ表現)

- **可読性の優先（ブログ/アーカイブ等）**: 長文コンテンツ（`/blog`など）では、読者の可読性を最優先にする。『Web Interface Guidelines』に従い、適切な行間（`leading`）、文字幅（`max-w-prose`）、文字サイズ（`text-base` からスケール）を設定すること。
- **レイアウトの堅牢性**: 画像のレスポンシブな振る舞いや、長いコードブロックがコンテナを突き破らないためのオーバーフロー対策（`overflow-x-auto` 等）を必ず適用し、レイアウト崩れを未然に防ぐこと。

## 4. Constraint over Creativity (創造性よりも制約)

- LLMは、「新しくて派手なUI」を勝手に発明してはならない。すでにあるコンポーネントライブラリや `globals.css` のユーティリティを組み合わせることで成立するかを最初に検討し、どうしても表現できない場合のみ拡張を提案すること。
