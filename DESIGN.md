# DESIGN

## 1. 原則

このリポジトリのUI/UXは、訪問者が一目で「プレミアム」だと感じる洗練された体験を提供することを目的とします。
シンプルなMinimum Viable Product (MVP) のような見栄えではなく、ダイナミックで豊かなデザインを採用します。

## 2. デザインシステムとタイポグラフィ

- **TailwindCSSベース**: 全てのスタイリングはTailwindCSSのユーティリティクラス（トークン）を用いて行い、マジックナンバーやインラインスタイルのハードコードを避けます（Token-first Styling）。
- **タイポグラフィ**: デフォルトのブラウザフォントは避け、モダンなフォント（Inter, Roboto, Outfitなど）を利用します。読みやすさ（可読性）を最優先にし、適切な行間とコントラストを維持します。
- **色彩とテーマ**: 汎用的な色（純粋な赤、青、緑）は避け、HSLベースの調和の取れたカラーパレットを使用します。サイト全体で洗練されたダークモードをサポートし、滑らかなグラデーションを活用します。

## 3. インタラクションとアニメーション

- **ダイナミックな体験**: ユーザーの操作（ホバー、クリック）に対して、微細かつ心地よいマイクロアニメーション（Hover effects, Micro-animations）を実装します。
- **応答性**: コンポーネントは状態の変化に対して視覚的なフィードバックを即座に返し、サイトが生きているかのように感じさせる設計とします。

## 4. Web Interface Guidelines (アクセシビリティ)

- **Semantic HTML**: 適切なHTML5セマンティック要素（`<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<footer>`）を使用します。
- **SEOベストプラクティス**: 1ページにつき1つの `<h1>` を配置し、適切な階層構造 (`<h2>`, `<h3>`) を維持します。
- **インタラクティブ要素**: すべてのインタラクティブな要素には、ブラウザテストやアクセシビリティツールのために、一意で説明的な ID (`id`) を付与します。
